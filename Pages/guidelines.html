📘 Contribution Guidelines

Build Better UI With Clean Standards

Follow these guidelines to maintain consistency, quality, and scalability across UIverse components.

💻 Code Style

  • ✔ Use semantic HTML structure
  • ✔ Keep CSS modular and reusable
  • ✔ Avoid inline styles
  • ✔ Follow consistent naming conventions (kebab-case)

🎨 UI Consistency

  • ✔ Maintain spacing consistency (padding/margin)
  • ✔ Use existing color variables
  • ✔ Ensure responsive design for all components
  • ✔ Keep typography aligned with Poppins font system

🧩 Component Rules

  • ✔ Each component should be independent
  • ✔ No dependency on external frameworks
  • ✔ Reusable across multiple pages
  • ✔ Keep components lightweight

🌱 Contribution Workflow

  • ✔ Fork the repository
  • ✔ Create a feature branch
  • ✔ Follow naming conventions
  • ✔ Submit a clean pull request
